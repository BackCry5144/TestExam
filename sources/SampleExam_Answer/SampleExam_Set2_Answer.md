**1. 정답: C**

* **해설:** **Orchestration Layer**는 최상위 레이어입니다. **End-User** 레이어 모듈이 상위 레이어를 참조하는 것은 "Upward Reference"로, 아키텍처 원칙에 어긋납니다.

**2. 정답: B**

* **해설:** 순환 참조(A↔B) 해결의 정석은 공통 기능을 추출하여 **하위 레이어(Core/Foundation)** 모듈로 옮기고 양쪽에서 이를 참조하게 하는 것입니다.

**3. 정답: A**

* **해설:** **Service Action**은 Weak Dependency를 가집니다. 따라서 인터페이스가 변경되지 않는 한, 로직 수정 후 배포해도 참조하는 모듈을 다시 배포할 필요가 없습니다.

**4. 정답: B**

* **해설:** **Core Layer**는 비즈니스 로직, 엔티티, 재사용 가능한 비즈니스 UI 블록 등을 포함하는 핵심 계층입니다.

**5. 정답: D**

* **해설:** Public Server Action은 다른 모듈에서 사용 시 특별한 데이터 타입 제한이 없습니다. (단, Service Action은 트랜잭션 공유 제한이 있음)

**6. 정답: C**

* **해설:** **Group By**를 적용하면 결과 리스트에는 그룹화된 기준 속성과 집계 결과(Count, Sum 등)만 남으며, 그룹화되지 않은 개별 속성()은 접근할 수 없습니다.

**7. 정답: B**

* **해설:** `Expand Inline` 파라미터는 SQL문에 직접 삽입되므로 **SQL Injection** 위험이 큽니다. 반드시 `EncodeSql()` 함수를 사용해야 합니다.

**8. 정답: D**

* **해설:** 인덱스는 조회 속도를 높이지만, 생성/수정 시 인덱스 테이블도 갱신해야 하므로 쓰기 성능을 저하시킬 수 있습니다. 따라서 무분별하게 많이 만드는 것은 좋지 않습니다.

**9. 정답: C**

* **해설:** 외부 API 결과와 DB 데이터를 결합하는 등 **복잡한 가공**이 필요한 경우 Aggregate 대신 **Data Action**을 사용해야 합니다.

**10. 정답: B**

* **해설:** `Count`는 전체 데이터 개수(), `Length`는 `MaxRecords` 설정에 따라 실제 가져온 개수()를 나타냅니다.

**11. 정답: B**

* **해설:** **Static Entity**는 환경(Dev, Prod) 간에 동일한 **Identifier(Id)**를 보장하기 위해 사용합니다.

**12. 정답: B**

* **해설:** **Timer**의 기본 타임아웃 시간은 **20분**입니다.

**13. 정답: A**

* **해설:** 대량 처리 시 `CommitTransaction`을 호출하면 트랜잭션 로그 포화를 막고, 타임아웃 발생 시 이미 처리된 부분은 저장되어 재처리가 효율적입니다.

**14. 정답: C**

* **해설:** **BPT**의 Automatic Activity에서 에러 발생 시, 시스템은 일정 대기 후 **자동으로 재시도(Retry)**를 수행합니다.

**15. 정답: A**

* **해설:** 프로세스를 명시적으로 시작하려면 **Process Launch Action**을 사용합니다.

**16. 정답: B**

* **해설:** 서버 트랜잭션 롤백은 DB에만 영향을 주며, 브라우저 메모리에 있는 **클라이언트 변수값**은 원래대로 돌아가지 않고 유지됩니다.

**17. 정답: C**

* **해설:** 루프 내에서 개별 `Create` 호출은 느립니다. **SQL Bulk Insert** 등을 사용하는 것이 성능상 최적입니다.

**18. 정답: C**

* **해설:** 권한이 없는 화면에 익명 사용자가 접근하면 시스템은 자동으로 **로그인 화면**으로 리다이렉트합니다.

**19. 정답: B**

* **해설:** `Internal Access Only`는 지정된 **IP 대역** 내에서만 접근을 허용하는 보안 설정입니다.

**20. 정답: C**

* **해설:** `CheckRole` 함수는 클라이언트와 서버 사이드 로직 **모두**에서 사용 가능합니다.

**21. 정답: B**

* **해설:** 현재 로그인한 사용자의 ID는 세션에서 관리하는 `GetUserId()` 함수를 통해 가져오는 것이 가장 안전합니다.

**22. 정답: B**

* **해설:** **Placeholder**는 부모 화면에서 블록 내부의 특정 영역에 원하는 콘텐츠를 동적으로 채울 수 있게 해줍니다.

**23. 정답: A**

* **해설:** `OnReady`는 화면의 **DOM 요소가 생성 완료**된 직후에 실행됩니다.

**24. 정답: B**

* **해설:** `Empty Text`는 변수값이 **Null**이거나 유효하지 않을 때 드롭다운에 표시되는 기본 문구입니다.

**25. 정답: B**

* **해설:** 옆으로 밀어서 버튼을 표시하는 기능은 OutSystems UI의 **List Action** 패턴입니다.

**26. 정답: C**

* **해설:** Strong Reference인 Server Action의 파라미터 변경은 참조 중인 모듈에 **컴파일 에러**를 유발합니다.

**27. 정답: B**

* **해설:** `Is Auto Number`는 **Integer** 계열 데이터 타입에서만 설정 가능합니다.

**28. 정답: B**

* **해설:** 1:1 관계의 자동 생성은 **Wrapper Action** 패턴을 통해 하나의 트랜잭션으로 묶어 처리하는 것이 정석입니다.

**29. 정답: B**

* **해설:** 에러 로그 및 모니터링은 **Service Center**의 Monitoring 탭에서 확인합니다.

**30. 정답: B**

* **해설:** 오래 걸리는 작업은 **비동기(BPT/Timer)**로 넘겨 UX를 개선하는 것이 좋습니다.

**31. 정답: B**

* **해설:** 화면 이동과 데이터 전달은 **Link** 위젯을 통해 수행합니다.

**32. 정답: B**

* **해설:** JS 노드 내외부의 값 교환은 노드의 **Input/Output Parameter**를 통해 매핑합니다.

**33. 정답: B**

* **해설:** Aggregate 결과 정렬은 **Sorting 탭**에서 속성을 추가하여 설정합니다.

**34. 정답: C**

* **해설:** 입력 시 즉시 반응하려면 Input 위젯의 **On Change** 이벤트를 사용합니다.

**35. 정답: B**

* **해설:** **Site Property**는 런타임에 값을 변경하여 애플리케이션의 동작(예: 점검 모드)을 제어하기에 적합합니다.

**36. 정답: B**

* **해설:** **Otherwise(Default)** 경로는 모든 조건이 일치하지 않을 때 실행됩니다.

**37. 정답: B**

* **해설:** Input 위젯의 `Variable` 속성에는 입력 데이터를 저장할 **변수**를 연결해야 합니다.

**38. 정답: A**

* **해설:** **Structure**는 메모리 상의 데이터 틀일 뿐, DB 테이블을 생성하지 않습니다.

**39. 정답: D**

* **해설:** `All Exceptions`는 이름 그대로 모든 유형의 예외를 처리할 수 있는 최상위 핸들러입니다.

**40. 정답: B**

* **해설:** Public Server Action 호출 시 별도 설정이 없다면 호출자의 **트랜잭션에 포함**됩니다.

**41. 정답: B**

* **해설:** Reactive Web은 화면 이벤트 핸들러인 **On Pull to Refresh**를 통해 해당 기능을 구현합니다.

**42. 정답: A**

* **해설:** `Ignore` 규칙은 부모 삭제를 허용하지만 자식 데이터를 남겨두므로 **고아 데이터** 문제를 야기합니다.

**43. 정답: B**

* **해설:** Client Action 간의 호출은 별도의 서버 통신 없이 브라우저 내에서 **동기적**으로 실행됩니다.

**44. 정답: B**

* **해설:** 조건부 UI 노출(권한 확인 등)에는 **If 위젯**이 가장 적합합니다.

**45. 정답: A**

* **해설:** 데이터 로딩 상태 확인은 Aggregate의 `IsDataFetched` 속성을 활용합니다.

**46. 정답: B**

* **해설:** 외부 시스템 연동(Connector)은 재사용성을 위해 **Foundation Layer**에 위치시킵니다.

**47. 정답: B**

* **해설:** `Only With`는 두 테이블 모두에 값이 있는 경우만 가져오는 **Inner Join**입니다.

**48. 정답: B**

* **해설:** `Raise Exception` 발생 시 즉시 실행이 중단되고 해당 예외를 처리할 **핸들러**로 점프합니다.

**49. 정답: B**

* **해설:** 식별자(Id) 변수의 기본 초기값은 `NullIdentifier()`입니다.

**50. 정답: B**

* **해설:** `OnDestroy`는 화면을 떠나거나 블록이 제거될 때 리소스를 해제하기 위해 사용합니다.
